Zadanie 2: Wizytówki na każdą okazję

W zadaniu 1 stworzyliśmy klasę, która pozwala nam tworzyć wizytówki zawierające imię, nazwisko i telefon. Wyobraźmy sobie teraz, że potrzebujemy także innego rodzaju wizytówek, które zawierają dodatkową informację - numer konta bankowego. Nie chcemy jednocześnie rezygnować z możliwości tworzenia starych wizytówek.

Rozszerzone wizytówki powinny drukować numer konta w dodatkowej linii i wyświetlać specjalną ramkę:

$$$$$$$$$$$$$$$$$$
$ Piotr Budynek                    $                                                                                                                
$ tel. 123456789                   $
$ 67 1234 5678 0000 0000 1234 5678 $
$$$$$$$$$$$$$$$$$$

Do realizacji zadania wykorzystaj mechanizm dziedziczenia.

  1. Stwórz klasę BankCard rozszerzającą Card.
  2. Zadeklaruj prywatne pole bankAccount typu String.
  3. Dodaj akcesor i mutator dla utworzonego pola.
  4. Utwórz konstruktor dla wszystkich atrybutów wizytówki. Wykorzystaj konstruktor klasy nadrzędnej.
  5. Rozszerz odpowiednią metodę klasy nadrzędnej, aby umożliwić wypisywanie numeru konta. Którą metodę wystarczy rozszerzyć? Co należy zmienić w klasie nadrzędnej by taka operacja była możliwa?
  6. Dokonaj odpowiednich modyfikacji w klasie bazowej tak, aby zmienić rodzaj wyświetlanej ramki. Wizytówki “zwykłe” powinny mieć ramkę złożoną z *, a wizytówki bankowe z $. Postaraj się również w tym przypadku skorzystać z dziedziczenia.

Przydatne informacje

Dziedziczenie

W programowaniu często skupiamy się na tym, by kod programu był zwięzły i nie zawierał powtarzających się instrukcji. Podstawowym sposobem na ponowne użycie kodu jest oczywiście definiowanie i wywoływanie metod (funkcji). Na poziomie klas Java również oferuje mechanizm, pozwalający wykorzystać nam już istniejący kod. Mechanizm ten nazywamy dziedziczeniem.

Powróćmy do przykłądu z klasą Point, określającą punkt w przestrzeni dwuwymiarowej (x,y):

public class Point {
   private double x;
   private double y;

   public Point(double x, double y) {
      this.x = x;
      this.y = y;
   }

   public double getX() {
      return x;
   }

   public void setX(double x) {
      this.x = x;
   }

   public double getY() {
      return y;
   }

   public void setY(double y) {
      this.y = y;
   }

   public double distance() {
      return Math.sqrt(x*x + y*y);
   }
}

Załóżmy, że chcemy teraz stworzyć Point3D, który będzie określał punkt w przestrzeni trójwymiarowej (x,y,z). Zauważmy, że klasa ta będzie w dużej mierze identyczna jak klasa Point, z tą różnicą że znajdzie się w niej dodatkowy atrybut z i jego obsługa. Zamiast kopiować kod klasy Point możemy zadeklarować, że klasa Point3D rozszerza klasę Point. Służy do tego słowo kluczowe extends. Definicja nowej klasy będzie wyglądała następująco:

public class Point3D extends Point {
...
}

Oczywiście chcemy by Point3D zawierał dodatkowy atrybut. Dodajemy więc odpowiednie pole i akcesory:

public class Point3D extends Point {
   private double z;

   public double getZ() {
      return z;
   }

   public void setZ(double z) {
      this.z = z;
   }
}

W tym momencie klasa Point3D zawiera zarówno wszystkie atrybuty, jak i metody klasy Point. Nie możemy jednak odwoływać się do nich w dowolny sposób. Kluczową rolę pełnią tu modyfikatory dostępu. W klasach rozszerzajacych możemy korzystać tylko z pól i metod, które w klasie bazowej zostały zadeklarowane jako public lub protected.

W przypadku metod możemy również nadpisywać zachowania klasy bazowej (o ile są one metodami public lub protected!). W naszym przykładzie chcemy na przykład na nowo zdefiniować działanie metody distance() żeby uwzględnić współrzędną z. W tym celu wystarczy dopisać ją w klasie Point3D:

public class Point3D extends Point {
   private double z;

   public double getZ() {
      return z;
   }

   public void setZ(double z) {
      this.z = z;
   }

   public double distance() {
      double x = getX();
      double y = getY();
      return Math.sqrt(x*x + y*y + z*z);
   }
}

Zwróćmy uwagę, że w rozszerzonym distance() odwołaliśmy się do metod klasy bazowej - nie mogliśmy jednak bezpośrednio użyć pól x i y, gdyż są one prywatne.
Możemy również odwołać się do oryginalnej wersji metody distance(). W tym celu powinniśmy skorzystać ze specjalnej referencji super. Działa ona podobnie jak this, z tą różnicą że wskazuje na obiekt klasy bazowej. W tej wersji rozszerzona metoda distance() wyglądałaby tak:

   public double distance() {
      double xyDistance = super.distance();
      return Math.sqrt(xyDistance*xyDistance + z*z);
   }

Ponieważ klasa Point zawierała konstruktor z dwoma argumentami, Point3D powinno skorzystać z niego w swoim konstruktorze. Tu również możemy użyć super(), tym razem odwołując się do nadrzędnego konstruktora. Uwaga: wywołanie super(...) zawsze musi być zamieszczone w pierwszej linijce konstruktora:

public class Point3D extends Point {
   private double z;

   public Point3D(double x, double y, double z) {
      super(x,y);
      this.z = z;
   }

   public double getZ() {
      return z;
   }

   public void setZ(double z) {
      this.z = z;
   }

   public double distance() {
      double xyDistance = super.distance();
      return Math.sqrt(xyDistance*xyDistance + z*z);
   }
}
